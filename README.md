# دفتر — ردیاب زنده دارایی‌های دیجیتال

اپلیکیشنی برای رصد بازار ارزهای دیجیتال، ساخته‌شده با **Next.js (App Router)**،
**React**، **TypeScript**، **Tailwind CSS** و **TanStack Query**. قیمت‌ها، جستجو و
فیلترها همگی از API عمومی [CoinGecko](https://www.coingecko.com/en/api) تأمین می‌شوند —
بدون نیاز به کلید API.

رابط کاربری کاملاً فارسی و راست‌به‌چپ (RTL) است.

## راه‌اندازی

```bash
npm install
npm run dev
```

سپس آدرس `http://localhost:3000` را باز کنید. نیازی به فایل `.env` نیست — چون از سطح
رایگان و عمومی CoinGecko مستقیماً از سمت کلاینت استفاده می‌شود.

> سطح رایگان CoinGecko محدودیت نرخ درخواست دارد (تقریباً ۱۰ تا ۳۰ درخواست در دقیقه).
> اگر پیام «محدودیت درخواست» دیدید، چند ثانیه صبر کنید — کش‌کردن هوشمند TanStack Query
> باعث می‌شود این اتفاق در استفاده‌ی عادی به‌ندرت رخ بدهد.

## چه چیزهایی این پروژه نشان می‌دهد

**دریافت و کش دیتا (TanStack Query)**
- کلیدهای کوئری بر اساس صفحه/دسته/شناسه ساختاریافته‌اند (`["markets", { page, category }]`)
  تا هر درخواست یکتا جداگانه کش شود.
- `staleTime` و `gcTime` طوری تنظیم شده‌اند که درخواست‌های یکسان طی ۳۰ ثانیه از کش
  سرو بشن، و `placeholderData: keepPreviousData` هنگام صفحه‌بندی جدول را پر نگه می‌دارد
  به‌جای نمایش حالت لودینگ.
- جستجو یک زنجیره‌ی دو مرحله‌ای است: ابتدا `/search` متن آزاد را به شناسه‌ی ارز تبدیل
  می‌کند، سپس کوئری دوم (که فقط وقتی شناسه‌ای وجود دارد فعال می‌شود) دیتای کامل بازار
  همان شناسه‌ها را می‌گیرد.

**جستجو و Debounce**
- هوک `useDebounce` تایپ کاربر را ۴۰۰ میلی‌ثانیه به تأخیر می‌اندازد تا زمانی که مکث
  کند، نه با هر ضربه کلید — یعنی درخواست فقط بعد از توقف تایپ ارسال می‌شود.

**فیلترینگ ترکیبی**
- نوع دارایی (دسته)، بازه قیمت و جهت تغییرات (صعودی/نزولی) همراه با یک کلید بازه زمانی
  (۱ساعت / ۲۴ساعت / ۷روز / ۳۰روز) ترکیب می‌شوند که تعیین می‌کند فیلتر و مرتب‌سازی روی
  کدام فیلد درصدی اعمال شود — همه‌ی این‌ها سمت کلاینت و روی داده‌ی دریافتی با
  `useMemo` انجام می‌شود.

**طراحی ریسپانسیو**
- یک ساختار جدول مشترک در دسکتاپ به‌صورت جدول کامل و در موبایل به‌صورت کارت تک‌ستونه
  نمایش داده می‌شود (`hidden sm:table-cell` و ردیف‌های مخصوص موبایل).
- نمودارها (اسپارک‌لاین‌ها و نمودار ناحیه‌ای پنل جزئیات) با Recharts داخل
  `ResponsiveContainer` ساخته شده‌اند تا در هر عرض صفحه‌ای درست بازچینی شوند.

## ساختار پروژه

```
app/
  layout.tsx        — فونت‌ها، متادیتا، QueryClientProvider، تنظیم dir="rtl"
  page.tsx           — استیت سطح صفحه: فیلترها، صفحه‌بندی، حالت جستجو/مرور
  providers.tsx       — تنظیمات کلاینت TanStack Query
  globals.css
components/
  TickerTape.tsx      — نوار قیمت زنده و اسکرول‌شونده (سرصفحه)
  SearchBar.tsx        — فیلد جستجوی debounce‌شده
  FilterBar.tsx         — کنترل‌های نوع / قیمت / بازه زمانی / مرتب‌سازی
  CoinTable.tsx          — پوسته‌ی جدول + حالت‌های لودینگ/خالی/خطا
  CoinRow.tsx             — ردیف دسکتاپ و کارت موبایل برای هر ارز
  Sparkline.tsx            — روند ۷ روزه به‌صورت خط کوچک درون‌خطی
  PriceChart.tsx            — نمودار ناحیه‌ای پنل جزئیات با تب‌های بازه زمانی
  CoinDetailDrawer.tsx       — پنل کشویی جزئیات با آمار و نمودار
  Pagination.tsx
lib/
  api.ts              — توابع فراخوانی CoinGecko
  types.ts             — تایپ‌های TypeScript مشترک برای پاسخ‌های API
  utils.ts              — قالب‌بندی قیمت، درصد و اعداد فشرده
  hooks/
    useDebounce.ts
    useCoins.ts          — تمام هوک‌های TanStack Query (بازار، جستجو، دسته‌ها، نمودار)
```

## نکات طراحی

هویت بصری («دفتر») به‌جای یک داشبورد تیره‌ی عمومی، به خودِ موضوع نزدیک شده: یک ثبت
زنده‌ی بازار. رنگ برنجی گرم (`#D4A24E`) برای عناصر تعاملی، فونت مونوی IBM Plex برای
تمام اعداد (تا ستون قیمت‌ها مثل یک دفتر حساب واقعی زیر هم بچینند)، و یک نوار قیمت
اسکرول‌شونده به‌عنوان عنصر امضادار صفحه.

برای متن فارسی از فونت **Vazirmatn** استفاده شده (هم برای عنوان‌ها و هم متن، فقط با
وزن‌های متفاوت)، و اعداد و قیمت‌ها عمداً با ارقام لاتین و فونت مونو نمایش داده می‌شوند
تا خوانایی و هم‌ترازی ستون‌های عددی حفظ شود — رویکردی رایج در رابط‌های مالی فارسی.

## ساده‌سازی‌های آگاهانه (در محدوده‌ی این نمونه)

- فیلترهای بازه قیمت و جهت تغییرات فقط روی *صفحه‌ی فعلاً بارگذاری‌شده* اعمال می‌شوند،
  نه کل بازار — چون endpoint عمومی CoinGecko از فیلتر سمت سرور روی این فیلدها پشتیبانی
  نمی‌کند. نسخه‌ی واقعی/تولیدی این فیلترها را به یک بک‌اند با دیتاست ایندکس‌شده منتقل
  می‌کند.
- بدون کلید API یا لایه‌ی احراز هویت، چون مستقیماً از endpointهای عمومی CoinGecko
  استفاده می‌شود.
